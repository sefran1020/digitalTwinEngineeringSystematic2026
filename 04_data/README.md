# Data — corpus, extraction, and derived tables

## Bibliography
- `corpus.bib` — the 65 included primary studies (clean author–year keys).
- `corpus_zotero.csv` — the Zotero export of the corpus (note: accented characters
  are byte-corrupted in this CSV; prefer `corpus.bib` for labels).
- `references_master.bib` — 193 deduplicated references (corpus + supporting
  literature) used to build the manuscript's reference list.

## `extraction/` — theme-conditional extraction
Each study activated only the extraction templates of the themes it was coded
under; every critical variable uses a controlled vocabulary (NR = not reported,
NA = not applicable, UNCERTAIN) and double verification with a verbatim quotation.

| File | Contents |
|---|---|
| `datos_largo.csv` | Long-format extraction: one row per (study, variable) with value, supporting quote, and location. |
| `articulos.csv` | One row per study with descriptive metadata. |
| `activacion.csv` | Theme-activation matrix (which themes T1–T8 each study activated). |
| `extraccion.json` / `.jsonl` | Structured extraction payloads. |
| `trazabilidad.jsonl` | Extraction provenance/audit trail. |

## `derived_tables/` — analysis outputs (generated by `../05_analysis/`)
- `paper_features.csv` — one row per study with the binary/categorical attributes
  used throughout (environment, Kritzinger maturity, AI, pedagogy, design, etc.).
- `capa1_*` — Layer 1: per-theme descriptive profiling with reporting rates.
- `capa2_*` — Layer 2: 0–6 evidence-quality composite, score distribution, and
  threshold-sensitivity analysis.
- `capa3_*` — Layer 3: theme co-occurrence and configuration analyses.
- `capa4_*`, `capa4b_*`, `capa4c_*`, `capa4d_*` — Layer 4 (exploratory relational):
  attribute associations (φ with bootstrap CIs), centrality, capability-by-evidence
  gap-map grid, and multiple-correspondence-analysis coordinates/inertia.
- `TA_evidencia_fuerte.csv`, `TB_arquitecturas_referencia.csv` — synthesis Tables
  1 and 2 (strong/moderate evidence; reference-architecture configurations).
- `citas_map.csv` — maps document keys to citation labels and `.bib` keys.
- `resumen.json` — summary counts.

## Theme → objective mapping
O1 = T1 (practical infrastructure) + T2 (enabling architectures) + T3 (AI
convergence); O2 = T4 (Industry 4.0/5.0 competencies) + T5 (pedagogical design);
O3 = T6 (empirical evidence); O4 = T7 (institutional value) + T8 (barriers).
